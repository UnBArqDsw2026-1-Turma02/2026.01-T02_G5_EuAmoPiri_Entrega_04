# Módulo 04 — Cadastro de Novo Local (Frontend)

Documento de reutilização de software para **RF04** (morador cadastra novo local) no **frontend** Eu Amo Piri. Validações server-side, persistência e upload de fotos estão em [backend/07.LocaisMorador.md](/ArquiteturaReutilizacao/backend/07.LocaisMorador.md).

---

## 1. O que foi implementado

| RF | Página / componente | Integração |
|----|---------------------|------------|
| RF04 | `CreatePlacePage` — formulário de cadastro de local | `placeAdaptor.createPlace` → `POST /places` (multipart/form-data) |
| RF04 | Upload de fotos com preview e arrastar-e-soltar | `URL.createObjectURL` para preview local; File[] enviados via `FormData` |
| RF04 | Overlay de sucesso/erro | Feedback imediato antes de navegar para `/locais` ou `/perfil` |

---

## 2. Bibliotecas e padrões reutilizados

| Componente | Origem | Papel |
| --- | --- | --- |
| **react-hook-form** | npm | Validação declarativa (nome, endereço, categoria, descrição obrigatórios) |
| **FormField** (molecule) | Implementação própria | Campo com label + input/textarea + erro; reutilizado de `EditPlacePage` e `LoginPage` |
| **Button** (atom) | Implementação própria | Botões de ação, cancelamento e overlays |
| **placeAdaptor.js** | *Adapter* | Monta `FormData`, chama API; fallback mock em `LOCAL_PLACES` + `sessionStorage` |
| **resolveMediaUrl** (`utils/mediaUrl.js`) | Implementação própria | Converte caminhos relativos do backend em URLs utilizáveis em `<img>` |
| **EditPlacePage.module.css** (baseStyles) | Implementação própria | Estilos de card, grid e overlays reutilizados sem duplicação — *CSS Module sharing* |
| **ProtectedRoute** `role=morador` | Implementação própria | Impede acesso por turistas ou usuários não autenticados — Guard Pattern |
| **React Router** | npm | `useNavigate` (Cancelar) + `Link` (overlays de sucesso) |

---

## 3. Rastreabilidade

| Critério | Evidência frontend |
|----------|--------------------|
| RF04 — morador preenche nome, endereço, categoria, descrição e pelo menos 1 foto | `CreatePlacePage` + validação `photos.length < 1` com `role="alert"` |
| RF04 — upload de 1 a 3 fotos com preview | Zona de drop, `URL.createObjectURL`, remoção com `revokeObjectURL` |
| RF04 — feedback de sucesso/erro | Overlay com destino "Ver locais" ou "Voltar ao perfil" |
| RF04 — rota protegida para morador | `ProtectedRoute requiredRole="morador"` em `AppRoutes.jsx` |

Backend correspondente: [07.LocaisMorador](/ArquiteturaReutilizacao/backend/07.LocaisMorador.md).

Documento detalhado: [rf04-cadastro-local](/docs/rf04-cadastro-local.md).

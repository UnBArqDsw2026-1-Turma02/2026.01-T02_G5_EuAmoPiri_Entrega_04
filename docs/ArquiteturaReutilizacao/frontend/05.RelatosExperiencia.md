# Módulo 05 — Relatos de Experiência (Frontend)

Documento de reutilização de software para **RF05** (publicar relato), **RF08** (editar/excluir relato) e **RF09** (painel do turista) no **frontend** Eu Amo Piri. Endpoints e validações server-side estão em [backend/04.RelatosExperiencia.md](/ArquiteturaReutilizacao/backend/04.RelatosExperiencia.md).

---

## 1. O que foi implementado

| RF | Página / componente | Integração |
|----|---------------------|------------|
| RF05 | `ExperienceForm` em `PlaceDetailPage` | `experienceAdaptor.createExperience` → `POST .../experiences` |
| RF08 | `EditExperiencePage` | Reuso de `ExperienceForm` com `defaultValues`; `updateExperience` / `deleteExperience` |
| RF09 | `ProfilePage` — seção turista | `experienceAdaptor.fetchMyExperiences` → `GET /auth/me/experiences` |

---

## 2. Bibliotecas e padrões reutilizados

| Componente | Origem | Papel |
| --- | --- | --- |
| **react-hook-form** | npm | Validação client-side antes do submit (complementa RNF01–03 do backend) |
| **ExperienceForm** (organismo) | Implementação própria | *Template* reutilizado em criar e editar (RF05 + RF08) |
| **experienceAdaptor.js** | *Adapter* | Traduz formulário ↔ `multipart/form-data` da API |
| **React Router** | npm | Navegação `/locais/:id/relatos/:id/editar` |
| **Vitest** | npm | Testes de `ProfilePage`, fluxos de edição |

---

## 3. Rastreabilidade

| Critério | Evidência frontend |
|----------|-------------------|
| RF05 — turista publica relato na página do local | `PlaceDetailPage` + `ExperienceForm` |
| RF08 — editar/excluir próprio relato | `EditExperiencePage`, modal em `ProfilePage` |
| RF09 — painel com relatos do turista | `ProfilePage` lista via `fetchMyExperiences` |

Backend correspondente: [04.RelatosExperiencia](/ArquiteturaReutilizacao/backend/04.RelatosExperiencia.md).

Documento detalhado de edição: [rf08-edicao-relatos](/requisitos/rf08-edicao-relatos.md).

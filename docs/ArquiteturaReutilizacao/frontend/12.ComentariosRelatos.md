# Módulo 12 — Comentários e Relatos na Página de Detalhe (Frontend)

Documento de reutilização de software para **RF12** (visualização e publicação de relatos na página de detalhe de um local) no **frontend** Eu Amo Piri. Endpoints de criação e listagem estão em [backend/04.ComentariosReacoes.md](/ArquiteturaReutilizacao/backend/04.ComentariosReacoes.md) e [backend/08.RelatosExperiencia.md](/ArquiteturaReutilizacao/backend/08.RelatosExperiencia.md).

---

## 1. O que foi implementado

| RF | Página / componente | Integração |
|----|---------------------|------------|
| RF12 | `PlaceDetailPage` — lista de relatos com `CommentCard` | `experienceAdaptor.fetchExperiencesByPlace` → `GET /places/:id/experiences` |
| RF12 | Formulário inline de novo relato para turistas autenticados | `experienceAdaptor.createExperience` → `POST /places/:id/experiences` |
| RF12 | `CommentsModal` — modal com todos os relatos | Reutiliza `CommentCard`; abre ao clicar em "Carregar Mais Comentários" |

---

## 2. Bibliotecas e padrões reutilizados

| Componente | Origem | Papel |
| --- | --- | --- |
| **CommentCard** (inline) | Implementação própria | Exibe autor, data, avaliação, texto e reações; reutilizado em preview e modal |
| **ExperienceForm** (organism) | Implementação própria | Formulário de relato reutilizado inline sem navegação para outra página |
| **StarRating** (atom) | Implementação própria | Exibe avaliação do relato em modo leitura |
| **Avatar** (atom) | Implementação própria | Foto do autor com fallback por iniciais |
| **AuthContext** / `useAuth` | Implementação própria | `isTurista` controla visibilidade do formulário; não autenticados veem link para login |
| **experienceAdaptor** | *Adapter* | Busca e cria relatos; fallback mock quando API indisponível |

---

## 3. Rastreabilidade

| Critério | Evidência frontend |
|----------|--------------------|
| RF12 — relatos listados na página do local | `PlaceDetailPage` + `fetchExperiencesByPlace` carregados no mount |
| RF12 — turista autenticado pode publicar relato inline | `showForm` state + `ExperienceForm` + `createExperience`; novo relato adicionado ao topo |
| RF12 — usuários não autenticados veem mensagem com link para login | Condicional `!isAuthenticated` na seção de comentários |
| RF12 — visualização completa em modal | `CommentsModal` com todos os relatos ao clicar "Carregar Mais Comentários" |

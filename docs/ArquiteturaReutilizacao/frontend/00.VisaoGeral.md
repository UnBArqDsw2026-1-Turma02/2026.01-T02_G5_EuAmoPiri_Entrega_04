# Reutilização de Software no Frontend — Visão Geral

Documento de síntese do módulo **Reutilização de Software** no escopo do **frontend** React da aplicação Eu Amo Piri. Esta pasta contém **apenas** artefatos de interface, adaptadores e bibliotecas consumidas pelo Vite/React — sem duplicar a documentação de API ou persistência (ver [backend/](/ArquiteturaReutilizacao/backend/00.VisaoGeral.md)).

---

## 1. Contexto do frontend

O frontend é uma **SPA** em **React 18 + Vite**. A equipe organizou páginas em `pages/`, componentes reutilizáveis em `presentation/` (Atomic Design), integração HTTP em `infra/adaptor/` e autenticação em `context/AuthContext.jsx`.

A reutilização concentra-se em **bibliotecas COTS** (React Router, react-hook-form, Vitest, react-leaflet) e em **padrões de projeto** (*Adapter*, *Guard*, composição atômica), reservando implementação própria para adaptadores de contrato API e fluxos de domínio de Pirenópolis.

## 3. Padrões transversais no frontend

| Padrão | Onde | Problema que resolve |
| --- | --- | --- |
| **Adapter** | `placeAdaptor.js`, `experienceAdaptor.js` | Desacopla páginas da forma real da API |
| **Guard** | `ProtectedRoute` | Autorização por papel antes de renderizar página |
| **Facade** | `authFacade` / `AuthContext` | Interface única de sessão para toda a SPA |
| **Atomic Design** | `presentation/atoms`, `molecules`, `organisms` | Composição reutilizável de UI |

# Módulo 03 — Cadastro, Autenticação e Perfil do Usuário (Frontend)

Documento de reutilização de software para **RF01** (cadastro de novo usuário), **RF02** (login/logout) e **RF03** (gestão de perfil) no **frontend** Eu Amo Piri. Lógica de JWT e hashing estão em [backend/02.Autenticacao.md](/ArquiteturaReutilizacao/backend/02.Autenticacao.md); persistência de perfil em [backend/03.PerfilArmazenamento.md](/ArquiteturaReutilizacao/backend/03.PerfilArmazenamento.md).

---

## 1. O que foi implementado

| RF | Página / componente | Integração |
|----|---------------------|------------|
| RF01 | `SignupPage` — formulário de cadastro com seleção de papel | `authAdaptor.register` → `POST /auth/register` |
| RF02 | `LoginPage` — formulário de login | `authAdaptor.login` → `POST /auth/login` |
| RF01/02 | `AuthContext` — estado global do usuário | `localStorage` (token JWT) + `GET /auth/me` |
| RF03 | `ProfilePage` — modo leitura e modo edição de perfil | `AuthContext.updateProfile` → `PUT /auth/me` |
| RF03 | Atualização de senha inline | Validação local + `PUT /auth/me/password` |

---

## 2. Bibliotecas e padrões reutilizados

| Componente | Origem | Papel |
| --- | --- | --- |
| **react-hook-form** | npm | Validação client-side (nome, e-mail, senha, papel, campos de perfil) antes do submit |
| **FormField** (molecule) | Implementação própria | Campo com label + input/textarea + mensagem de erro; reutilizado em cadastro, login e perfil |
| **Button** (atom) — variantes `danger`, `secondary`, `rust`, `primary` | Implementação própria | Botões semânticos: vermelho=destruição, marrom=edição, ferrugem=exclusão, verde=criação |
| **Avatar** (atom) | Implementação própria | Foto de perfil com fallback por iniciais + upload comprimido via canvas |
| **Badge** (atom) | Implementação própria | Exibe papel do usuário com cor semântica |
| **AuthContext** / `useAuth` | Implementação própria | Provê `user`, `login`, `logout`, `register`, `updateProfile`, `isMorador` — *Singleton* via Context API |
| **authAdaptor.js** | *Adapter* | Traduz chamadas dos componentes ↔ endpoints REST; fallback mock em desenvolvimento |
| **React Router** | npm | Redirecionamento pós-cadastro e proteção de `/perfil` |
| **ProtectedRoute** | Implementação própria | Guard de rota por papel — reutilizado em todas as rotas privadas |

## 3. Rastreabilidade

| Critério | Evidência frontend |
|----------|--------------------|
| RF01 — usuário preenche nome, e-mail, senha e papel | `SignupPage` + `react-hook-form` com regras `required`, `pattern`, `minLength` |
| RF01 — seleção de papel (turista / morador) | `<select>` com opções `turista` e `morador`; papel persistido no token |
| RF02 — login com e-mail e senha | `LoginPage` + `authAdaptor.login` → token JWT salvo no `localStorage` |
| RF02 — logout | `AuthContext.logout` remove token e reseta estado global |
| RF03 — usuário edita nome, e-mail, profissão, contato, bio e foto | `ProfilePage` modo edição + `updateProfile` |
| RF03 — atualização de senha com validação | Seção "CADASTRAR NOVA SENHA" com verificação de igualdade e comprimento mínimo |

Backend correspondente: [02.Autenticacao](/ArquiteturaReutilizacao/backend/02.Autenticacao.md) · [03.PerfilArmazenamento](/ArquiteturaReutilizacao/backend/03.PerfilArmazenamento.md).

Documento detalhado: [rf03-perfil](/docs/rf03-perfil.md).

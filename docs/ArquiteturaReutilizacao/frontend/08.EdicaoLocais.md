# Módulo 08 — Edição e Exclusão de Locais (Frontend)

Documento de reutilização de software para **RF07** (morador edita e exclui locais que cadastrou) no **frontend** Eu Amo Piri. Validações server-side e persistência estão em [backend/07.LocaisMorador.md](/ArquiteturaReutilizacao/backend/07.LocaisMorador.md).

---

## 1. O que foi implementado

| RF | Página / componente | Integração |
|----|---------------------|------------|
| RF07 | `EditPlacePage` (`/morador/locais/:id/editar`) — formulário pré-populado | `placeAdaptor.fetchPlaceById` → `GET /places/:id`; `updatePlace` → `PUT /places/:id` |
| RF07 | `ProfilePage` — `MoradorSections` com botões "Editar Local" e "Excluir Local" | `placeAdaptor.deletePlace` → `DELETE /places/:id` |
| RF07 | Modal de confirmação de exclusão inline | Remove item da lista otimisticamente sem recarga de página |

---

## 2. Bibliotecas e padrões reutilizados

| Componente | Origem | Papel |
| --- | --- | --- |
| **react-hook-form** + `reset()` | npm | Pré-popula formulário com dados do local carregado; validação de campos obrigatórios |
| **FormField** (molecule) | Implementação própria | Campo com label + input + erro; reutilizado de `CreatePlacePage` e `LoginPage` |
| **Button** / **Spinner** (atoms) | Implementação própria | Botões de ação e feedback visual de carregamento |
| **placeAdaptor** (`fetchPlaceById`, `updatePlace`, `deletePlace`) | *Adapter* | Abstrai fonte de dados (mock → API real) sem alterar os componentes |
| **EditPlacePage.module.css** | Implementação própria | Grid de formulário reutilizado por `CreatePlacePage` como `baseStyles` |
| **ProtectedRoute** `role=morador` | Implementação própria | Impede acesso de turistas a `/morador/locais/:id/editar` — Guard Pattern |
| **useParams** + **useNavigate** (React Router) | npm | Extrai `:id` da URL; redireciona para `/perfil` após salvar |
| Modal overlay inline | Implementação própria | Mesmo padrão visual do `CommentsModal` de `PlaceDetailPage` |

---

## 3. Rastreabilidade

| Critério | Evidência frontend |
|----------|--------------------|
| RF07 — morador edita campos do local | `EditPlacePage` + `useForm({ defaultValues })` pré-populado via `fetchPlaceById` |
| RF07 — feedback de sucesso com redirecionamento | Mensagem inline + `setTimeout(() => navigate('/perfil'), 1200)` |
| RF07 — morador exclui local com confirmação | Modal em `MoradorSections`; `deletePlace` chamado; item removido da lista sem reload |
| RF07 — erro de exclusão não fecha o modal | `deleteErr` exibido dentro do modal; usuário pode tentar novamente |

Backend correspondente: [07.LocaisMorador](/ArquiteturaReutilizacao/backend/07.LocaisMorador.md).

Documento detalhado: [rf07-edicao-locais](/docs/rf07-edicao-locais.md).

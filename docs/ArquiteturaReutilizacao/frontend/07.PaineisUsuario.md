# Módulo 07 — Painéis do Turista e do Morador (Frontend)

Documento de reutilização de software para **RF09** (gestão de contribuições — turista) e **RF10** (acompanhamento de relatos — morador) no **frontend** Eu Amo Piri. Os endpoints consumidos estão em [backend/04.RelatosExperiencia.md](/ArquiteturaReutilizacao/backend/04.RelatosExperiencia.md) e [backend/03.LocaisMorador.md](/ArquiteturaReutilizacao/backend/03.LocaisMorador.md).

---

## 1. O que foi implementado

| RF | Papel | UI | Chamadas |
|----|-------|-----|----------|
| RF09 | Turista | `ProfilePage` — seção “Avaliações cadastradas” | `GET /auth/me/experiences` |
| RF10 | Morador | `ProfilePage` — locais do morador + relatos por local | `fetchMoradorPlaces()` + `fetchExperiencesByPlace` por local |

A equipe **compôs no frontend** o painel RF10 (sem tela admin dedicada): lista locais do morador e, para cada um, carrega relatos públicos — espelhando a composição REST documentada no backend.

---

## 2. Bibliotecas e padrões reutilizados

| Componente | Origem | Papel |
| --- | --- | --- |
| **ProfilePage** | Atomic Design | Painel unificado por `accountType` (turista vs morador) |
| **placeAdaptor.fetchMoradorPlaces** | *Adapter* | `GET /places?moradorId` |
| **experienceAdaptor** | *Adapter* | Relatos por `placeId` |
| **AuthContext** | *Facade* | Identidade e papel do usuário logado |
| **React Router** | npm | `/perfil` protegido |

---

## 3. Rastreabilidade

| RF | Critério | Evidência frontend |
|----|----------|-------------------|
| RF09 | Turista vê todos os seus relatos | `ProfilePage` + `fetchMyExperiences` |
| RF10 | Morador vê relatos nos seus locais | `ProfilePage` morador + loop de locais/relatos |
| RNF04 | Histórico não público por userId | UI só no `/perfil` autenticado |

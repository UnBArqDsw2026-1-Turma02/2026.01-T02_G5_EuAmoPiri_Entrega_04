# Módulo 06 — Consulta e Filtros de Locais (Frontend)

Documento de reutilização de software para **RF06** (consulta de locais) e **RF15** (filtros dinâmicos sobre catálogo Google + comunitário) no **frontend** Eu Amo Piri. A API de listagem está documentada em [backend/09.ConsultaPublica.md](/ArquiteturaReutilizacao/backend/09.ConsultaPublica.md) e o sync Google em [backend/06.SincronizacaoGooglePlaces.md](/ArquiteturaReutilizacao/backend/06.SincronizacaoGooglePlaces.md).

---

## 1. O que foi implementado

| RF | Componente | Comportamento |
|----|------------|---------------|
| RF06 | `PlacesPage.jsx` | Carrega `fetchPlaces()`; exibe grid/mapa de locais |
| RF06 | Campo de busca | Filtro client-side por nome (`search` state) |
| RF06 | Chips de categoria | Filtro instantâneo Cachoeira / Restaurante / Pousada |
| RF06 | `PlaceDetailPage` | Detalhe + relatos públicos do local |
| RF15 | Mesma `PlacesPage` | Catálogo unificado (morador + Google) sem endpoint paralelo |
| RF15 | BDD filtro “Cachoeiras” | `useMemo` oculta outras categorias sem reload |

---

## 2. Bibliotecas e padrões reutilizados

| Componente | Origem | Papel |
| --- | --- | --- |
| **placeAdaptor.js** | *Adapter* | `fetchPlaces`, `resolveMediaUrl` — URLs relativas `/places/:id/cover` |
| **PlaceCard** (molécula) | Atomic Design | Card reutilizado na listagem |
| **react-leaflet** | npm | Mapa na listagem — *Adapter* sobre Leaflet |
| **useMemo** (React) | React | Filtros RF06/RF15 sem round-trip ao servidor |
| **Vitest** | npm | `PlacesPage.test.jsx` |

---

## 3. Rastreabilidade — RF15 BDD 2

| Critério | Evidência |
|----------|-----------|
| Visitante vê locais importados + comunitários | `GET /places` via `placeAdaptor` |
| Filtro “Cachoeiras” instantâneo | `PlacesPage` — `category` state + `useMemo` |
| Sem recarregar página | Estado local React; sem `window.location` |

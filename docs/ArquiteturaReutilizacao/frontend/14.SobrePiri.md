# Módulo 14 — Seção "Sobre Piri" (Frontend)

Documento de reutilização de software para **RF14** (página informativa sobre Pirenópolis) no **frontend** Eu Amo Piri. Conteúdo estático — sem endpoint de backend dedicado.

---

## 1. O que foi implementado

| RF | Página / componente | Integração |
|----|---------------------|------------|
| RF14 | `SobrePiriPage` — três seções: Origem, História e Atualidade | Conteúdo estático; sem chamada de API |
| RF14 | Desktop: todas as seções visíveis simultaneamente | CSS Media Query `min-width: 768px` reverte `.secaoOculta` para `display: block` |
| RF14 | Mobile: navegação por abas (pills) com botões Anterior / Próximo e indicador "1 / 3" | `useState` + classe `.secaoOculta` (`display: none`) |

---

## 2. Bibliotecas e padrões reutilizados

| Componente | Origem | Papel |
| --- | --- | --- |
| **useState** (React) | React | Controla `secaoAtiva` sem biblioteca externa de tabs |
| **Link** (react-router-dom) | npm | Botão "← Início" para navegação SPA sem reload de página |
| **CSS Modules** | Implementação própria | Escopo local de estilos sem conflito com outros componentes |
| **Design tokens** (`tokens.css`) | Implementação própria | `--color-brown`, `--radius-full` e demais tokens reutilizados; sem valores hard-coded |
| Array `SECOES` + mapa `SECAO_LABELS` | Implementação própria | Centraliza metadados das seções evitando repetição — *Data-driven UI* |

---

## 3. Rastreabilidade

| Critério | Evidência frontend |
|----------|--------------------|
| RF14 — página exibe informações sobre Pirenópolis | `SobrePiriPage` com seções Origem, História e Atualidade |
| RF14 — navegação por seções no mobile | Pills de aba + botões Anterior/Próximo; botões `disabled` nos extremos da lista |
| RF14 — layout responsivo | Desktop: todas as seções visíveis; mobile: uma seção por vez com indicador de progresso |

Sem backend correspondente — conteúdo inteiramente estático no frontend.

Documento detalhado: [rf14-sobre-piri](/docs/rf14-sobre-piri.md).
